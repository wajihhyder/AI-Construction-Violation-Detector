# Screenshots & diagrams

Drop the following images into this folder (and `docs/`) — they are referenced by name from the
project READMEs and will render automatically once added.

| File | What to capture |
| :--- | :--- |
| `docs/architecture.png` | The styled system architecture diagram (optional — a Mermaid version already renders in the root README). |
| `docs/screenshots/home_page.png` | Landing page (project intro + complaint tracking). |
| `docs/screenshots/citizen_report.png` | Citizen reporting wizard (district selection + image upload). |
| `docs/screenshots/citizen_result.png` | AI analysis result — detected floors and compliance verdict. |
| `docs/screenshots/authority_dashboard.png` | Authority map dashboard (reports on a map). |
| `docs/screenshots/authority_report_detail.png` | Report detail view with the printable violation notice. |
| `docs/screenshots/admin_users.png` | Admin user-management screen. |
| `docs/screenshots/model_predictions.png` | A grid of annotated YOLO floor detections on sample images. |

**Tip:** keep screenshots at a consistent width (e.g. 1280px) and use PNG for UI, JPG for photos.
